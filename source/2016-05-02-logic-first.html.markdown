---
title: Logic First
date: 2016-05-02 09:37 EDT
tags:
---
The most important thing in a software system is it's logic.  What to do, when and how is the problem software solves.  Unfortunately, the logic of a software system becomes less and less clear over time to the developers that work on it.

As a piece of software grows in terms of lines of code and people complexity grows along with it.  The decisions that drive the logic of the system begin to get buried underneath thousands of lines of code.  What should happen vs. what does happen in a given situation turns into a forensic investigation rather than an answer to even the simplest of questions.

Developers are poor at documentation and achieving full test coverage.  They overfocus on development tools and express logic in convoluted programming languages.  I think the only way forward is to flip the script and change what it means to write a software system logic-first.

Logic development should take precedence over programming.  Code will come and go but it's the logic of the system that matters.
