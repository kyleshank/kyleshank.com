date: 2018-07-24 12:23 EDT

---

I'm now writing more Python than Ruby code.  After 13 years of preferring Ruby as my go to scripting language I've come around to liking Python more and more.  

Python has become the first choice for server side client work recently.  We are mostly using it for server side backend APIs to power React frontends and mobile apps.

I have started to prefer serverless deployments using Amazon's API Gateway and Lambda services.  Lambda supports Python along with Java, Go, C# and Javascript.  Clients are starting to prefer this approach and again, Python fits the bill.

For those reasons and the quality of its third party libraries Python has replaced Ruby for me.
