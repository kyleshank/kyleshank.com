title:  "Swipe Stopper is now Open Source"
date:   2014-07-31 16:19:00
---
[I have open sourced the code for Swipe Stopper.](https://github.com/formal-method/SwipeStopper-iOS)

Why? I like to publish code.  It's the only certain way to ensure a piece of work lives on.

Swipe Stopper isn't a hit app.  It has less than 2000 downloads.  I don't know how to market it and there will never really be a way to profit from the idea.

The code does have value though.  At the very least its a good example of my work.  The most value though is for those just getting started with iOS and want to see what a simple and complete application looks like.  I have always learned best by example and there are very few examples of complete mobile applications for people to learn from.

I have released 5 open source projects in the past month.  My goal going forward to open source all of my work.  The next big project will be open sourcing Cloudgram which consists of an iOS application, Android application and Rails backend.  I really think the Cloudgram codebase would be a great reference example for someone wanting to build a multi-platform mobile app with a web front and backend.
